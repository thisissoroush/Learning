## 🗂️ Topics Covered

Based on the current book summaries, this section covers:

- 🧠 **Pragmatic philosophy** — taking responsibility, fighting entropy, owning your career and craft
- 🏗️ **Design principles** — ETC (Easier to Change), DRY (Don't Repeat Yourself), orthogonality, decoupling
- 🔧 **Essential tools** — plain text, shell mastery, editor fluency, version control, debugging
- 🛡️ **Defensive programming** — design by contract, assertions, crash-early strategies, resource management
- 🔀 **Flexibility** — decoupling code, event-driven design, transformations, externalizing configuration
- ⚡ **Concurrency** — temporal coupling, shared state pitfalls, actor model
- ✍️ **Coding habits** — programming by coincidence, refactoring, test-driven development, naming
- 📋 **Before the project** — requirements gathering, solving the right puzzle, working with stakeholders
- 👥 **Pragmatic teams** — automating everything, delivering well, delighting users, team pride
